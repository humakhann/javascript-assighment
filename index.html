ANS- 1: == (Double equals): This checks if two things are equal in value, but it might perform type coercion, meaning it can convert the operands to the same type before making the comparison. For example: 5 == '5' would return true because it converts the string '5' to a number and compares their values. 2: === (Triple equals): This also checks if two things are equal in value, but it's stricter; it doesn’t perform type coercion. It checks both the value and the type of the operands. For example: 5 === '5' would return false because it doesnt convert types; it directly compares a number to a string and sees they're different types, so it returns false. In simple terms, == checks if values are the same after potential type conversions, while === checks for both value and type without doing any conversions. ANS-parseInt in JavaScript is a function that takes a string as input and tries to convert it into an integer (a whole number). In simple words, it helps you change a string that looks like a number into an actual number that you can use for calculations or comparisons. FOR ex- let numberString = "123"; let number = parseInt(numberString); console.log(number); // This will output: 123 It takes a string like "123" and turns it into the number 123. This is useful when you have user inputs or data from sources where numbers are represented as strings, and you need to use them as actual numbers in your JavaScript code. Please enter a valid number."); } This script uses prompt to take input from the user, then converts the input to a number using parseInt. It checks if the number is valid and then uses the modulo operator % to determine if the number is even or odd. If the remainder of dividing the number by 2 is 0, it's even; otherwise, it's odd.
